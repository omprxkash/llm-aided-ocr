The LLM-Aided OCR Pipeline

A seven-step pipeline turns any scanned PDF into clean, structured text — fully configurable, chunked for large documents, and scored for quality.

Phase 1 — Extraction

PDF to Raw Text
via Tesseract OCR

Quick setup ›
PDF → Image Conversion
Each PDF page is rasterised to a high-resolution image using pdf2image and Poppler. Page ranges and skip counts are fully configurable.
Image Pre-processing
OpenCV applies Otsu thresholding and dilation to improve character contrast before handing off to Tesseract.
Concurrent OCR Extraction
Pages are processed in parallel via ThreadPoolExecutor, cutting wall-clock time for large documents significantly.

Phase 2 — Correction

Intelligent Chunking
& LLM Error Correction

Choose your LLM ›
Context-Aware Chunking
Large documents are split at paragraph and sentence boundaries into overlapping 8,000-character chunks so no context is lost across chunk boundaries.
OCR Error Correction Prompt
Each chunk is sent with a structured prompt that instructs the LLM to fix split words, misread characters (rn → m), and stray page numbers while preserving all original meaning.
Async Concurrent Processing
When using OpenAI as the API-based LLM, all chunks are dispatched concurrently via asyncio.gather() and re-ordered after completion.

Phase 3 — Output

Markdown Formatting
& Quality Assessment

Run the pipeline ›
Structured Markdown
The corrected text is passed through a second LLM prompt that converts it to proper Markdown — headings, lists, bold, tables — while removing duplicate content and spurious LLM artifacts.
Automated Quality Score
A final LLM call compares the original OCR output against the processed result, returning a numeric score (0–100) and a human-readable explanation of improvements.
Dual File Output
Both the raw OCR (__raw_ocr_output.txt) and the LLM-corrected result (_llm_corrected.md) are always saved for comparison.

LLM Providers

Cloud or Fully Local

Switch between the OpenAI API and a fully local Llama model with a single environment variable. No code changes needed.

Default
🤖
OpenAI
gpt-4o-mini · gpt-4o

Cloud API with async concurrent chunk processing. Fast, accurate, and ideal for production workflows.

API_PROVIDER="OPENAI"
🦙
Local LLM
Llama 3.1 8B · GGUF

Runs fully offline via llama-cpp-python. Auto-downloads the model. GPU-accelerated with CUDA fallback to CPU.

USE_LOCAL_LLM=True
.env — Configuration
# Choose provider
USE_LOCAL_LLM=False
API_PROVIDER="OPENAI"

# OpenAI
OPENAI_API_KEY="sk-proj-…"
OPENAI_COMPLETION_MODEL="gpt-4o-mini"
# Local Llama (offline, only if USE_LOCAL_LLM=True)
DEFAULT_LOCAL_MODEL_NAME="Llama-3.1-8B…"
LOCAL_LLM_CONTEXT_SIZE_IN_TOKENS=2048
USE_VERBOSE=False

Command-Line Interface

Automate your
OCR pipeline

The CLI wraps the full pipeline with fine-grained control over page ranges, hallucination filtering, Markdown output, and embedding storage.

●
--max-pages / --skip-pages
Process any subset of a document's pages
●
--no-markdown
Output plain text instead of structured Markdown
●
--threshold / --test-filtering
Tune and test hallucination similarity filtering
●
--check-english
Validate that extracted text is valid English
llm-aided-ocr-cli.py
# Basic usage
$ python src/llm-aided-ocr-cli.py document.pdf

# Process pages 2–10 only
$ python src/llm-aided-ocr-cli.py document.pdf \
--skip-pages 1 --max-pages 9

# Plain text, validate English
$ python src/llm-aided-ocr-cli.py document.pdf \
--no-markdown --check-english

# Tune hallucination filter
$ python src/llm-aided-ocr-cli.py document.pdf \
--threshold 0.35 --test-filtering
Deployment

Up in minutes,
anywhere you run containers

A pre-built Docker image bundles Tesseract, Poppler, Python 3.12, and all dependencies. GPU passthrough is configured out of the box.

Quick Start (pip)
# 1. Clone and enter
$ git clone <repo-url> && cd llm-aided-ocr

# 2. Install dependencies
$ pip install -r requirements.txt

# 3. Configure
$ cp .env.example .env && nano .env

# 4. Run
$ python src/llm_aided_ocr.py
Docker Compose (GPU)
# Start with GPU support
$ docker compose up -d

# Image: hotwa/llm_aided_ocr:latest
# Port: 3232 → SSH into container
# VRAM: 32 GB shared memory

# Run OCR on a file
$ docker exec llm_aided_ocr \
python src/llm_aided_ocr.py
